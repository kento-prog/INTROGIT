Gitの練習をします。
基本的なフローを学びます。

ファイルを変更したら
1.ステージングエリアへ追加 
   git add [ファイル名]
   git add .
2. リポジトリにメッセージをつけてコミット
   git commit
   コミットメッセージ
   1行目：変更内容の要約
   2行目：空行
   3行目：変更内容の理由

3. git status   現在の状況の確認
   git rm  削除
   git log  変更履歴の確認  --oneline;1行で表示する　   -p ファイル名: ファイルの差分表示
  
4. git diff; 変更差分の表示、add,commitする前に表示 
   git diff HEAD; ステージとコミットの差分
